Beta build 9 corrects the beta release pipeline so update detection and release notes stay consistent across published builds.

Updater And Feed Fixes

  • Preserved public access to Sparkle endpoints such as appcast.xml, latest.json, release notes, and DMG downloads.
  • Confirmed the live appcast now serves valid XML and points to the current beta artifact instead of the gated access page.
  • Shipped a strictly higher Sparkle build number so existing Beta 8 installs can finally detect and install the next update.

Release Process Fixes

  • Release notes are now published per build using paths like /beta/release-notes/1.0.0-beta.9.
  • The downloads site now derives the displayed beta number from the release-notes slug instead of mixing latest metadata with older notes content.
  • The GitHub release workflow now blocks same-build or lower-build republishes for the same version so Sparkle metadata stays monotonic.

Notes

  • If you installed an earlier Beta 8 artifact manually, this Beta 9 release is the first one that should appear as an in-app update.
  • The legacy version-only release-notes URL is still kept as an alias to the latest published build for compatibility.
