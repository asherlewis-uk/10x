Beta build 10 makes previews more dependable during launch, keeps project thumbnails up to date, and fixes a rollback issue so your earlier work history stays visible.

More Reliable Previews

  • Projects that open straight into their main screen are now much more likely to show a preview correctly on the first run.
  • The first captured preview is no longer as likely to disappear during startup, which should reduce blank or missing initial previews.
  • Screen detection during launch is more consistent, helping 10x attach the preview to the right view earlier in the process.

Cleaner Project Library

  • Project thumbnails on the Home screen now refresh as soon as a new preview is captured, so saved projects reflect the latest version of your app.
  • Rolling back to an earlier message now preserves the visible work history from that point instead of hiding older progress that should remain.

Notes

  • Existing projects will update their thumbnail after the next successful preview capture.
  • No migration or manual cleanup is required for this build.
